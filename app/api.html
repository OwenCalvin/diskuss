Diskuss API

This page shows the comprehensive list of request that are included in the Diskuss API.

Diskuss is a chat server that can host numerous people in any number of channels. It handles group chat along with private chat.

Information

This request returns informations about the server itself to anyone, without authentication.

Request

GET /info

Response

Key Value
version The version of the server.

Connection

In order to connect to the server, the client must request a GUID.

Request

POST /users/register/:nick
Key Value
nick The desired nickname

Response

Key Value
id The GUID linked to the client. As this is the only way to identify someone, this must never be lost.
nick The nickname linked to the client. If the desired nickname was unavailable, "_1" will be append to it. If thi is also unavailable, "_2" will be append and so on.

Whois

Anyone can ask the server informations about a connected user.

Request

GET /users/id/:id/whois/:nick
Key Value
id The client own GUID
nick The nickname of the user one wants to have informations about

Response

Key Value
id If the client asks for its own informations, its GUID is sent.
nick the nickname of the user about which one wants informations.

Error handling

Whatever the URI may be, if the resource or action asked is unavailable for some reason, a JSON containing a single key called error will be sent. The value of that entry is the error message.
